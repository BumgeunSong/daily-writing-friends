## ADDED Requirements

### Requirement: Drag and Drop Image Upload
사용자는 이미지 파일을 에디터 영역에 드래그 앤 드롭하여 업로드할 수 있어야 한다. 드래그 진입 시 시각적 피드백을 제공하며, 비이미지 파일은 거부한다.

#### Scenario: 이미지 파일을 에디터에 드롭
- **WHEN** 사용자가 이미지 파일(PNG, JPEG, GIF, WebP)을 에디터 영역에 드롭한다
- **THEN** 이미지가 처리(리사이즈/압축)된 후 Firebase Storage에 업로드되고, 에디터의 현재 커서 위치에 삽입된다

#### Scenario: 드래그 진입 시 시각적 피드백
- **WHEN** 사용자가 파일을 에디터 영역 위로 드래그한다
- **THEN** 에디터 영역에 하이라이트 스타일이 표시된다

#### Scenario: 드래그 이탈 시 피드백 제거
- **WHEN** 사용자가 파일을 에디터 영역 밖으로 드래그한다
- **THEN** 하이라이트 스타일이 제거된다

#### Scenario: 비이미지 파일 드롭 거부
- **WHEN** 사용자가 비이미지 파일(PDF, 텍스트 등)을 에디터에 드롭한다
- **THEN** 업로드가 거부되고 "이미지 파일만 업로드할 수 있습니다" 토스트가 표시된다

### Requirement: Paste Image Upload
사용자는 클립보드의 이미지를 에디터에 붙여넣어 업로드할 수 있어야 한다.

#### Scenario: 클립보드 이미지 붙여넣기
- **WHEN** 사용자가 클립보드에 이미지가 있는 상태에서 Ctrl+V/Cmd+V를 누른다
- **THEN** 클립보드의 이미지가 처리된 후 Firebase Storage에 업로드되고, 에디터의 현재 커서 위치에 삽입된다

#### Scenario: 텍스트 붙여넣기는 기존 동작 유지
- **WHEN** 사용자가 클립보드에 텍스트만 있는 상태에서 Ctrl+V/Cmd+V를 누른다
- **THEN** Quill의 기본 텍스트 붙여넣기 동작이 그대로 수행된다

### Requirement: Multi-file Upload
사용자는 파일 선택기에서 여러 이미지를 동시에 선택하여 업로드할 수 있어야 한다. 파일들은 순차적으로 처리되며, 실패한 파일은 건너뛴다.

#### Scenario: 여러 이미지 선택 및 순차 업로드
- **WHEN** 사용자가 파일 선택기에서 N개의 이미지를 선택한다
- **THEN** 각 이미지가 순차적으로 처리/업로드되고, 토스트에 "이미지 업로드 중... (M/N)" 형태로 진행 상황이 표시된다

#### Scenario: 부분 실패 시 계속 진행
- **WHEN** N개 이미지 중 일부가 업로드에 실패한다
- **THEN** 실패한 파일은 건너뛰고 나머지 파일의 업로드를 계속 진행한다. 완료 후 "X장 업로드 완료, Y장 실패" 요약 토스트가 표시된다

#### Scenario: Multi-file 삽입 위치
- **WHEN** 사용자가 여러 이미지를 선택하고 업로드 중 에디터에서 텍스트를 입력한다
- **THEN** 모든 이미지는 시퀀스 시작 시점의 커서 위치를 기준으로 순서대로 삽입된다 (사용자의 타이핑과 무관)

### Requirement: Large File Support
원본 파일 크기가 5MB를 초과하더라도 최대 20MB까지 허용하며, 리사이즈/압축 처리 후 업로드한다. 처리 후에도 5MB를 초과하면 거부한다.

#### Scenario: 5MB 초과 20MB 이하 이미지 업로드
- **WHEN** 사용자가 5MB~20MB 크기의 이미지를 업로드한다
- **THEN** 이미지가 리사이즈/압축 처리된 후 Firebase Storage에 업로드된다

#### Scenario: 20MB 초과 이미지 거부
- **WHEN** 사용자가 20MB를 초과하는 파일을 업로드한다
- **THEN** "파일이 너무 큽니다" 토스트가 표시되고 업로드가 거부된다

#### Scenario: 처리 후에도 5MB 초과 시 거부
- **WHEN** 이미지 처리(리사이즈/압축) 후에도 파일이 5MB를 초과한다
- **THEN** "처리 후에도 파일이 큽니다" 토스트가 표시되고 업로드가 거부된다

### Requirement: Editor Remains Usable During Upload
이미지 업로드 중에도 에디터는 잠기지 않으며, 사용자는 텍스트 입력을 계속할 수 있어야 한다.

#### Scenario: 업로드 중 텍스트 입력
- **WHEN** 이미지 업로드가 진행 중이다
- **THEN** 에디터는 잠기지 않고 사용자가 텍스트를 계속 입력할 수 있다. 진행 상황은 토스트로만 표시된다

### Requirement: HEIC Conversion Fallback
HEIC/HEIF 파일의 JPEG 변환이 실패하면, 원본 파일을 그대로 업로드 시도한다.

#### Scenario: HEIC 변환 성공
- **WHEN** 사용자가 HEIC 파일을 업로드하고 변환이 성공한다
- **THEN** JPEG으로 변환된 파일이 리사이즈 후 업로드된다

#### Scenario: HEIC 변환 실패 시 원본 업로드
- **WHEN** 사용자가 HEIC 파일을 업로드하고 heic2any 변환이 실패한다
- **THEN** 원본 HEIC 파일로 업로드를 시도한다 (사이즈 제한 내에서)

### Requirement: File Validation
업로드 전 파일 타입과 사이즈를 검증한다. 검증 로직은 순수 함수로 분리하여 테스트 가능해야 한다.

#### Scenario: 이미지 MIME 타입 허용
- **WHEN** `image/*` MIME 타입의 파일이 업로드된다
- **THEN** 파일 타입 검증을 통과한다

#### Scenario: HEIC 확장자 허용
- **WHEN** `.heic` 또는 `.heif` 확장자의 파일이 업로드된다 (MIME 타입이 `image/`로 시작하지 않더라도)
- **THEN** 파일 타입 검증을 통과한다

#### Scenario: 비이미지 파일 거부
- **WHEN** 이미지가 아닌 파일(PDF, 텍스트 등)이 업로드된다
- **THEN** "이미지 파일만 업로드할 수 있습니다" 토스트가 표시되고 업로드가 거부된다

### Requirement: Existing Toolbar Upload Preserved
기존 툴바 이미지 버튼을 통한 업로드 기능은 그대로 유지되어야 한다.

#### Scenario: 툴바 버튼으로 이미지 업로드
- **WHEN** 사용자가 툴바의 이미지 버튼을 클릭하고 파일을 선택한다
- **THEN** 이미지가 처리 후 업로드되고 에디터에 삽입된다 (기존과 동일한 동작)
