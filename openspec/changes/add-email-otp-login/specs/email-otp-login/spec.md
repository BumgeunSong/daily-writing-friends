## ADDED Requirements

### Requirement: 등록된 이메일로 OTP 코드 발송

시스템은 사용자가 입력한 이메일이 기존 등록 user의 이메일일 때만 6자리 OTP 코드를 발송해야 한다 (SHALL). 미등록 이메일에 대해서는 코드를 발송하지 않고 에러를 반환해야 한다.

#### Scenario: 등록된 이메일에 대한 OTP 발송 성공

- **WHEN** 사용자가 Google OAuth로 이미 가입한 이메일을 입력하고 "이메일로 인증 코드 받기"를 클릭
- **THEN** 시스템은 해당 이메일로 6자리 숫자 OTP를 발송하고, UI를 코드 입력 단계로 전환한다

#### Scenario: 미등록 이메일에 대한 발송 거부

- **WHEN** 사용자가 등록되지 않은 이메일을 입력하고 "이메일로 인증 코드 받기"를 클릭
- **THEN** 시스템은 OTP를 발송하지 않고, "등록되지 않은 이메일입니다." 에러 메시지를 노출하며, UI는 이메일 입력 단계에 머문다

### Requirement: OTP 코드 검증 및 세션 생성

시스템은 발송된 6자리 OTP가 올바를 때 사용자의 세션을 생성하고 SIGNED_IN 이벤트를 발생시켜야 한다 (SHALL). 잘못된 코드에 대해서는 세션을 생성하지 않고 에러를 노출해야 한다.

#### Scenario: 올바른 OTP 입력으로 로그인 성공

- **WHEN** 사용자가 메일로 받은 6자리 OTP를 정확히 입력하고 "로그인"을 클릭
- **THEN** 시스템은 Supabase 세션을 생성하고, `onAuthStateChange`가 SIGNED_IN 이벤트를 발생시키며, 사용자는 메인 화면으로 진입한다

#### Scenario: 잘못된 OTP 입력 처리

- **WHEN** 사용자가 잘못된 6자리 코드를 입력하고 "로그인"을 클릭
- **THEN** 시스템은 세션을 생성하지 않고, "인증 코드가 올바르지 않습니다." 에러 메시지를 노출하며, 입력란과 입력값은 유지된다

### Requirement: 같은 이메일의 Google 계정과 자동 identity linking

시스템은 OTP 로그인이 성공할 때 동일 이메일로 가입된 기존 Google OAuth user와 동일한 `user.id`를 가진 세션을 생성해야 한다 (MUST). 별도의 신규 user는 생성되지 않아야 한다.

#### Scenario: 동일 이메일의 기존 Google user와 동일 user.id

- **WHEN** Google OAuth로 가입한 user `U`의 이메일로 OTP 로그인을 완료
- **THEN** 생성된 세션의 `user.id`는 `U.id`와 동일하고, `auth.identities` 테이블에는 user `U`에 대해 `google` provider와 `email` provider 두 identity 행이 존재한다

### Requirement: 2단계 UI 흐름 (이메일 → 코드)

로그인 페이지는 OTP 흐름을 'email' 단계와 'code' 단계로 분리해 노출해야 한다 (SHALL). 사용자는 코드 단계에서 이메일 입력 단계로 되돌아갈 수 있어야 한다.

#### Scenario: 이메일 단계 UI

- **WHEN** 사용자가 로그인 페이지에 진입했고 OTP 흐름을 시작하지 않은 상태
- **THEN** 화면에는 구글 로그인 버튼, 구분선("또는"), 이메일 입력란, "이메일로 인증 코드 받기" 버튼이 표시된다

#### Scenario: 코드 단계 UI

- **WHEN** OTP 발송이 성공하여 코드 입력 단계로 전환됨
- **THEN** 화면에는 이메일이 표시된 안내 문구, 6자리 숫자 입력란(`inputMode='numeric'`, `maxLength=6`), "로그인" 버튼, "이메일 다시 입력하기" 텍스트 버튼이 표시된다

#### Scenario: 이메일 단계로 복귀

- **WHEN** 사용자가 코드 단계에서 "이메일 다시 입력하기"를 클릭
- **THEN** UI는 이메일 입력 단계로 복귀하고, 에러 메시지는 초기화되며, 이전에 입력한 이메일 값은 유지된다

### Requirement: 6자리 숫자 입력 가드

코드 입력란은 6자리 숫자만 허용하고, 6자리 미만일 때는 검증 요청을 차단해야 한다 (SHALL).

#### Scenario: 비숫자 입력 자동 제거

- **WHEN** 사용자가 코드 입력란에 영문자나 특수문자를 입력
- **THEN** 입력란은 숫자만 남기고 나머지 문자를 제거한다

#### Scenario: 6자리 미만 시 버튼 비활성화

- **WHEN** 사용자가 코드 입력란에 6자리 미만의 숫자를 입력한 상태
- **THEN** "로그인" 버튼은 disabled 상태이며 검증 요청이 발생하지 않는다
