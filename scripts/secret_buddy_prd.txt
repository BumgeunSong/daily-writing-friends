# Overview
비밀 친구(Secret Buddy) 기능은 활성 보드가 있는 모든 유저에게 1:1 순환 구조로 비밀 친구(known buddy)를 랜덤 할당하는 기능입니다. 본인은 자신의 known buddy만 알 수 있고, known buddy는 자신을 secret buddy로 둔 사용자를 모릅니다.

# Core Features
- 모든 활성 보드 유저에게 1:1 순환 구조로 buddy 랜덤 할당
- 본인은 자신의 known buddy만 알 수 있음
- known buddy는 자신을 secret buddy로 둔 사용자를 모름(비대칭)
- UserPage에서 known buddy 정보 표시
- BoardPage에서 secret buddy가 known buddy의 글을 볼 때 인스타그램 스타일의 프로필 액세서리(gradient border) 표시
- Firestore에 knownBuddy 필드 저장
- 1회성 allocateSecretBuddy.ts 스크립트 작성
- Remote Config에서 active_board_id fetch, 활성 유저만 대상으로 buddy 할당
- 실행 전 기존 buddy 관계 초기화(knownBuddy 필드 삭제)

# Data Models
- User: { id, name, avatarUrl, knownBuddy: userId }
- Board: { id, active: boolean }

# API & Scripts
- allocateSecretBuddy.ts: 활성 보드의 모든 유저를 대상으로 buddy 랜덤 할당
- Firestore: users 컬렉션에 knownBuddy 필드 업데이트
- Remote Config: active_board_id fetch

# UI/UX
- UserPage: 내 known buddy 정보 표시
- BoardPage: secret buddy가 known buddy의 글을 볼 때 프로필 액세서리(gradient border) 표시
- ProfileAccessory, KnownBuddyProfileAccessory, ComposedAvatar, PostUserProfile 등 컴포넌트 설계

# Development Roadmap
1. Firestore 데이터 모델 및 컬렉션 설계
2. allocateSecretBuddy.ts 스크립트 작성 및 테스트
3. UserPage/BoardPage UI 구현
4. ProfileAccessory 등 컴포넌트 리팩터링
5. QA 및 배포

# Risks and Mitigations
- buddy 할당 중 중복/누락 방지: 1:1 순환 구조, 랜덤 셔플 후 순환 할당
- 기존 buddy 관계 초기화: 스크립트 실행 전 knownBuddy 필드 삭제
- UI/UX 혼동 방지: 본인은 자신의 known buddy만 볼 수 있도록 제한
